`/go` 페이지는 Google Maps 실시간 렌더링으로 인해 모바일 LCP(8.1s)·TBT(480ms)가 특히 심각했습니다. JS 실행 시간 1.7초, 메인 스레드 작업 2.4초, 긴 작업이 모바일에서 11개나 발견됐습니다.

## 원인과 개선

[원본: docs/성능/성능개선/go-맵동적로딩.md, 담당 박소유, P0, 상태: 완료]

`@vis.gl/react-google-maps` SDK(~2MB)가 초기 번들에 포함되어 페이지 진입 즉시 다운로드·실행되면서 메인 스레드를 블로킹했습니다. LCP 대상인 지도 자체가 SDK 로드 완료 후에야 렌더링되는 구조였습니다.

세 가지 대안을 검토하고 모두 기각했습니다:
- mounted state + 스켈레톤 단독 — SDK가 여전히 초기 번들에 포함돼 TBT가 개선되지 않음
- IntersectionObserver로 스크롤 시 로드 — 맵이 전체 화면을 차지해 "스크롤로 뷰포트 진입"이라는 조건 자체가 성립하지 않음
- 정적 지도 이미지 대체 — 핀 클릭·드래그·줌이 핵심 기능이라 UX 저하

최종적으로 `next/dynamic(ssr:false)`로 `GoMapClient`를 lazy load해 Maps SDK를 초기 번들에서 완전히 분리하고, 구글맵 배경색(`#e5e3df`)과 필터칩/버튼 스켈레톤을 갖춘 `GoMapSkeleton`을 로딩 상태로 표시했습니다. 개선 후 수치는 배포 후 재측정 예정으로 아직 기록되지 않았습니다.
