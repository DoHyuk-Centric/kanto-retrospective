면접에서 "완벽했다"고 말하는 것보다, 무엇이 왜 남았는지 정확히 아는 게 더 신뢰를 준다고 생각해서 이 페이지를 따로 뒀습니다.

## 완성된 것

마켓플레이스 3종(중고거래·구인구직·렌탈) 목록/상세/작성/수정/삭제, 모임(Go) 구글 맵 기반 지역 모임과 그룹채팅, 실시간 채팅(1:1·그룹), 결제(Xendit 인보이스·에스크로·후기), 알림, 신고/모더레이션(AI 이미지 모더레이션 포함), 다국어(한/영/필) 완전 번역 동기화, AI 챗봇. 어드민은 대시보드·사용자/게시글/채팅 관리·감사 로그·운영 설정·권한 관리·결제 관리까지 갖췄습니다. 코드 청결도 측면에서는 TODO/FIXME 주석 0건, console.log 잔존 0건, i18n 번역 동기화 완전 일치(23개 최상위 키 기준)로 확인됐습니다.

## 미완성/한계로 남은 것

**Critical — 접근 시 404**: `/business` 라우트 하위 3개 페이지(`/business/company`, `/business/company/create`, `/business/dashboard`)에 `page.tsx` 파일이 없어 실제 접근이 불가능합니다. 컴포넌트 폴더와 레이아웃 구조는 존재하지만 실제 페이지 구현이 없는 상태입니다.

**High — 테스트 부재**: 프로젝트 전체에 테스트 파일이 단 한 개도 없습니다. 결제·인증 등 크리티컬 경로도 테스트가 적용돼 있지 않습니다. ([자세한 내용은 테스트 전략 페이지 참고](/strategy/테스트전략))

**Low — PWA/오프라인 미지원**: `manifest.json`, Service Worker가 없습니다.

**코드 품질 이슈**: 서비스 레이어에서 Supabase `error`를 확인하지 않고 `data`만 사용하는 패턴이 결제·칸토Go 서비스 여러 곳에 존재합니다.

**보안 이슈(평가 시점 기준, 이후 리뷰에서 부분 반영)**: 어드민 GET 엔드포인트(`api/admin/sanction-templates`, `api/admin/profanity-rules`) 인증 누락, 검색 파라미터(`keyword`) 미검증, PUT 작업 감사 로그 누락, 환경변수 시작 시 유효성 검증 없음.

**접근성**: Chatbot 버튼 aria-label 누락, Pagination `<nav>` aria-label 필요, SignupForm 에러 메시지의 `aria-describedby` 미연결(단, Header·ChatInput 등 대부분의 ARIA 구현은 양호하다고 평가).

**성능**: 이미지 `loading="eager"` 전면 적용, AI 공급자(Gemini·Groq·Cerebras) 동시 import로 인한 Cold Start 지연 가능성, 번들 분석 도구 미설정 — 이 중 다수는 이후 [성능개선](/performance) 작업으로 실제 대응됐습니다.

## 우선순위 액션 아이템(원문 기준)

P1(어드민 GET 인증, 비즈니스 페이지 구현/제거) → P2(에러 핸들링 통일, 파라미터 검증, 감사 로그) → P3(환경변수 검증, 접근성, 이미지 lazy) → P4(테스트 인프라, 번들 분석, PWA manifest) 순으로 정리돼 있습니다.

이 평가는 2026-06-26 시점의 스냅샷입니다. 이후 리뷰 사이클(9~16차)에서 관리자 인가 문제나 SEO, 성능 이슈 상당수가 실제로 개선됐지만, `/business` 페이지 미구현이나 팀 자체 테스트 인프라 부재처럼 이 문서를 작성하는 시점까지도 그대로 남아있는 항목이 있습니다. 무엇이 실제로 닫혔고 무엇이 아직 열려있는지는 각 섹션([기능구현](/features), [문제해결](/troubleshooting), [성능개선](/performance))에 근거를 남겨뒀습니다.
